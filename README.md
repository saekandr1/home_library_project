# home_library_project
# 📜 Система учёта книг «Домашняя библиотека» (Кейс №10)

Добро пожаловать в репозиторий моего итогового проекта. Это кроссплатформенное приложение на языке Python, созданное для удобного поиска и ведения учёта книг в домашней библиотеке.

Проект разрабатывался от простого текстового скрипта до полноценного графического объектно-ориентированного приложения со связанными справочниками.

---

## 🚀 Архитектурная эволюция проекта

В репозитории сохранены все три версии программы, отражающие этапы разработки:

1. **`library_app1.py` (Базовый уровень)** — консольная версия. Управление библиотекой происходит через ввод цифр в текстовом меню терминала. Данные сохраняются в базу данных `home_library_v1.db`, доступна простая выгрузка отчета.
2. **`library_app2.py` (Средний уровень)** — первая графическая версия с ярким дизайном. База данных здесь «плоская» (авторы и жанры пишутся каждый раз вручную строками, справочников ещё нет). Работает со своим изолированным файлом базы данных `home_library_v2.db`.
3. **`library_app3.py` (Продвинутый уровень)** — финальная объектно-ориентированная версия. Дизайн переведён в благородные пастельные тона. Реализована полноценная реляционная СУБД `home_library_v3.db` (3 связанные таблицы) и добавлены продвинутые UX/UI фишки.

---

## ✨ Главные фишки финальной версии (v3.0)

* **Реляционная база данных с каскадным удалением:** База данных SQLite разделена на три таблицы (`books`, `authors`, `genres`). На уровне СУБД настроено ограничение `ON DELETE CASCADE` — при удалении автора или жанра из справочника, все связанные с ними книги стираются автоматически, защищая базу от «мусора».
* **Ввод с подсказками:** Окошко с вариантами авторов и жанров открывается от первой же буквы по внутреннему вхождению слова.
* **Кнопки справочников «под рукой»:** Кнопки управления справочниками жанров и авторов (**📂**) перенесены прямо в карточку книги, справа от полей ввода.
* **Регистронезависимый поиск силами Python:** Чтобы избежать проблем с поиском русских букв на уровне СУБД, фильтрация перенесена в код Python с применением метода `.lower()`. Программа найдёт книгу независимо от того, как написан запрос: «сказка», «Сказка» или «СКАЗКА».
* **Управление с клавиатуры:** Поддерживается полноценная навигация кнопкой **Tab** и активация действий по нажатию **Enter**. Строка поиска мгновенно запускает фильтрацию по клавише **Enter** без использования мыши.
* **Автоматический демо-фонд (Data Seeding):** При самом первом запуске на пустой базе программа автоматически создаёт справочники и наполняет систему 37 известными книгами и 10 жанрами (включая детскую литературу), чтобы можно было сразу протестировать сортировку и отчеты.
* **HTML-отчётность:** Кнопка «HTML-отчет» генерирует стильную веб-страницу `library_report.html` с адаптивной табличной версткой, карточками статистики и точным временем создания среза данных.

## 🛠 Инструкция по запуску

Программа написана на чистом Python и использует исключительно встроенные библиотеки (`tkinter`, `sqlite3`, `os`, `datetime`). **Никакие дополнительные библиотеки скачивать не нужно (`pip install` выполнять не требуется).**

1. Склонируйте репозиторий или скачайте файлы проекта в одну папку.
2. Откройте терминал/командную строку в этой папке.
3. Выполните команду для запуска финальной версии:
   ```bash
   python library_app.py
   ```
   *(Или запустите любой другой файл, например `python library_app2.py`, чтобы посмотреть средний уровень реализации).*
